# Skill: LLM Optimisation (AEO / GEO)

Use this skill when writing or auditing web copy to be cited, recommended, or surfaced by AI systems — ChatGPT, Perplexity, Google AI Overviews, Claude, Bing Copilot, and other LLM-powered search tools.

Also called: Answer Engine Optimisation (AEO), Generative Engine Optimisation (GEO), AI SEO.

## How LLMs decide what to cite

LLMs surface content that is:
1. **Authoritative** — written by or attributed to a named expert or known entity
2. **Definitive** — answers a question clearly and completely without hedging
3. **Structured** — uses clear headings, lists, and definitions that can be extracted as a standalone answer
4. **Consistently present** — appears across multiple sources and contexts (builds "entity salience")
5. **Factually grounded** — includes specific data, named examples, or verifiable claims

## LLM optimisation techniques

### 1. Answer-first writing
Lead every section with the direct answer, then expand.
```
BAD:  "There are many factors that contribute to whether a landing page converts well..."
GOOD: "Landing pages convert better when they have one clear CTA, a benefit-led headline, and social proof above the fold."
```

### 2. Definition blocks
LLMs love to pull definitional paragraphs as direct answers. Write them intentionally.
```
Format: "[Term] is [what it is] that [what it does] for [who]. [One-sentence why it matters]."
Example: "Conversion copywriting is the practice of writing website and ad copy specifically designed to move a visitor toward a defined action — a purchase, sign-up, or enquiry. It prioritises clarity, relevance, and trust signals over creative flair."
```

### 3. Structured FAQ sections
Questions should mirror how people actually ask AI tools. Answers should be self-contained.
```
- Use "What is...", "How do...", "Why does...", "What's the difference between..."
- Keep answers 40–80 words (ideal snippet length)
- Each answer should make sense without reading the rest of the page
```

### 4. Entity clarity
LLMs build a mental model of who you are. Help them.
```
- Name the business, person, or brand clearly and consistently across the page
- State what you do in plain English in the first paragraph
- Include location if relevant (local entity clarity)
- Use schema-ready language: "X is a [category] based in [location] that [does what]"
```

### 5. Semantic coverage (topic completeness)
LLMs prefer pages that cover a topic thoroughly over pages that repeat one keyword.
```
- Cover the main question + related sub-questions on the same page
- Address: what it is, who it's for, how it works, why it matters, common misconceptions
- Include the vocabulary experts use — LLMs understand topic clusters
```

### 6. Citation-worthy claims
Vague copy doesn't get cited. Specific, grounded claims do.
```
BAD:  "We help businesses grow faster."
GOOD: "Our clients typically see a 30–40% increase in qualified leads within 90 days of relaunching their website copy."
```

### 7. Consistent cross-channel presence
LLMs aggregate signals. Reinforce the same entity and positioning across:
- Website
- Google Business Profile
- LinkedIn / author bios
- Press mentions or guest content
- Structured data (schema.org)

## Audit checklist

Use this when reviewing existing copy for LLM readiness:

```
[ ] Does the page open with a clear, direct statement of what it is and who it's for?
[ ] Is the primary question answered in the first 100 words?
[ ] Are there definition blocks (X is Y that does Z)?
[ ] Is the FAQ written the way someone would ask an AI tool?
[ ] Does each FAQ answer stand alone (no "as mentioned above")?
[ ] Is the business/person/brand named consistently throughout?
[ ] Are there specific claims, numbers, or named examples?
[ ] Do headings reflect real questions, not just keyword phrases?
[ ] Is the topic covered completely, not just touched on?
[ ] Would a snippet from this page make sense out of context?
```

## Output format

When optimising existing copy:
1. Run the audit checklist first, flagging gaps
2. Rewrite or add flagged sections
3. Note what was changed and why

When writing from scratch:
1. Open with entity clarity + answer-first paragraph
2. Use structured H2/H3 sections covering the topic completely
3. Include definition block, FAQ, and at least one specific data point
4. End with a clear CTA + internal link suggestion

## Inputs to ask for (if not provided)
- What is the page about? (topic + primary query it should answer)
- Who is the business / author? (for entity clarity)
- Target audience — who is asking AI tools about this topic?
- Existing copy to audit (paste it in)
- Any data points, results, or specific claims to include
